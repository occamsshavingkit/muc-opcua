# Implementation Plan: Server Nano Core Type System & Base Info CUs

**Branch**: `069-nano-type-system` | **Date**: 2026-07-13 | **Spec**: [spec.md](spec.md)
**Roadmap**: spec 005 — Server: Nano Core Type System & Base Info CUs

## Summary

Add ~16 static OPC UA node entries to `base_nodes.c` implementing the Core 2022
Server Facet type-system and Base Info conformance units at the nano level. Each
CU gates behind a `#ifdef MUC_OPCUA_CU_*` guard. 4 CUs are mandatory for nano
(default y), 12 are optional (default n), per OPC API `isOptional` flags.

## Technical Context

**Language/Version**: C11 (freestanding subset, C99-compatible)
**Primary Dependencies**: None (static tables only; no new files)
**Storage**: N/A (static const data in .rodata)
**Testing**: CMocka (`tests/unit/test_profile_surface.c`, `test_base_server_behaviour.c`)
**Target Platform**: Host (Linux) + Pico SDK cross-compile + `freestanding-core`
**Project Type**: C library (embedded OPC UA server)
**Performance Goals**: No runtime overhead — all nodes are static const
**Constraints**: Must not regress nano `.text` > 30 KB; each CU must compile out individually
**Scale/Scope**: ~16 CUs × ~3-8 static node entries each = ~50-100 new nodes
**OPC UA Normative References**: OPC-10000-3 §4 (AddressSpace), OPC-10000-5 (NodeIds)
**Target OPC UA Profile/Conformance Units**: Core 2022 Server Facet (id 1322) — nano minimum
**Conformance Status Target**: profile-targeting (CTT verification not yet done)

## Embedded Size Budget

**Flash Impact**: ~1-3 KB total for all 16 CUs (static node tables, pooled strings)
**RAM Impact**: 0 bytes (no dynamic allocations; all const data)
**Heap Use**: none
**Static Tables Added**: ~50-100 new `mu_node_t` entries (~120 bytes each) = ~6-12 KB worst case;
  practical estimate ~3 KB after string pooling
**Transport Buffers**: N/A
**Crypto Dependency Impact**: N/A
