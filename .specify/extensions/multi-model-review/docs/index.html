multi-model-review

Multi-model code review for Spec-Driven Development.

Write specs with one model, implement with another, and review with a different one. A Claude Code plugin built on top of GitHub Spec Kit.

Why multi-model?

A model reviewing its own output often rationalizes. A different model can catch different classes of issues, and this project makes that handoff cheap enough to do regularly.

  • separate model routing for spec authoring, implementation, subagent task slices, and review
  • portable markdown package instead of shared session state
  • compact-first packaging to reduce token cost
  • Codex CLI reviewers can use local OSS providers through --oss
  • confidence-gated findings for easier remediation

Detailed model specs use <model>[:<axis-a>][@<axis-b>]. Codex axes map to reasoning and speed; Claude axes map to context and workload.

Codex local providers

Codex CLI review and spec-author handoffs can run through local OSS providers such as Ollama or LM Studio when Codex is configured with oss_provider and invoked with --oss. Provider selection stays in Codex CLI configuration; the extension keeps the selected model string in handoff metadata.

codex exec --oss -m <local-model> --file .cross-review/packages/<pkg>/review-package.md > .cross-review/packages/<pkg>/review-report.md

Automatic subagent routing

The plugin ships Claude Code subagents for fast scouting, heavy planning, scoped implementation, and local review checks. When subagent_routing.mode is auto, task hints such as [route:scout] or [route:worker] let Claude Code pick the matching subagent and configured role model.

Compact-first review packages

The default package is no longer a full dump of spec.md, plan.md, tasks.md, and the raw diff. It now starts with focused summaries and high-signal excerpts inspired by RTK.

ProfileUse case
microtiny MCP-sized review loops
compactdefault balanced package
fullescalation pass when the reviewer needs more context

If a reviewer needs more information, it can say Context sufficiency: needs-full-package and the builder reruns packaging with --full or narrows the scope with --paths.

Install

Slash commands such as /plugin install run inside Claude Code, not at your OS shell.

/plugin install multi-model-review@multi-model-review

Or at the OS shell:

claude plugin install multi-model-review@multi-model-review
claude plugin list

Then run /reload-plugins or restart Claude Code.

Commands

InvocationPurpose
/multi-model-review:cross-review [init|status|models set] [--spec <model[:axis]@axis>] [--spec-heavy <model[:axis]@axis>] [--dev <model[:axis]@axis>] [--review <model[:axis]@axis>] [--subagents auto|off]configure model routing, subagent roles, reviewer, base ref, and default package profile
/multi-model-review:spec-handoff [slug|brief] [--spec-model <model[:axis]@axis>] [--heavy] [--plan] [--subagents auto|off]export the development spec or plan handoff
/multi-model-review:review-package [slug|task-id] [--review-model <model[:axis]@axis>] [--base <ref>] [--full] [--paths <glob,...>]export the compact reviewer handoff
/multi-model-review:apply-review [path] [--min-confidence N] [--subagents auto|off]ingest the review report and walk through fixes

Quick workflow

  1. Run /multi-model-review:cross-review init --spec codex-5.5:xhigh@normal --spec-heavy opus-4.7:1m@max --dev sonnet-4.6@high --review codex-5.5:high@normal --subagents auto.
  2. Run /multi-model-review:spec-handoff --spec-model codex-5.5:xhigh@normal or add --heavy for a large audit.
  3. Implement the feature with your configured implementation model and route clear slices to the matching subagent; silent upgrade is disabled by default.
  4. Run /multi-model-review:review-package --review-model codex-5.5:high@normal to create a compact package.
  5. Run the reviewer model on that package in another terminal.
  6. Run /multi-model-review:apply-review to apply accepted findings.
